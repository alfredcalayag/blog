I love pizza. I love wine. Put them together and you have the perfect dinner (by my standards). Put them together in Ruby and you can make a hash. Put them in JavaScript and you can make a _________.

Just as I started to get comfortable, the rug was swept under me. This week, we put Ruby aside and started learning the basics of JS. Okay, it wasn't that bad, and I had a great opportunity to observe some of the similarities and differences from Ruby. In particular, it was sad to see Hashes get the boot. They proved to be very useful for storing values in a 1:1 fashion, i.e. keys and values. In the world of JS, o where art thou, hash? Disguised in a different name, you say?... *drum roll* ......... 'OBJECTS'

Yes, hashes are to Ruby as objects are to JS. Join me at the table over some food, spirits, and discussion of the hash/object comparison. Back to the food, shall we.

I stumbled across Wine Folly in trying to find some fun practical hash examples (as well as dinner ideas before my wife gets home). Matching different wines with different pizzas... sounds delicious doesn't it? Let's store it in a hash while it's still hot! Recall how this would look like:

# In Ruby:
pairing_ruby = {sangiovese: 'pepperoni', ganache: 'margherita', syrah: 'sausage', zinfandel: 'hawaiian', chardonnay: 'white', malbec: 'barbeque chicken'}

// In JS:
var pairing_js = {sangiovese: 'pepperoni', ganache: 'margherita', syrah: 'sausage', zinfandel: 'hawaiian', chardonnay: 'white', malbec: 'barbeque chicken'}

The var in the JS example comes with initializing most variables and all apart of the syntax/documentation that comes with the language. I won't go into too much detail on that here, but will simply show the side-by-side Ruby-JS examples.

We can retrieve all of the contents by using the following code:

# In Ruby:
pairing_ruby

// In JS:
pairing_js

Both of these lines of code will return wine/pizza pairings to us.

Oh, how nice of you to bring some a bottle of Zin to the party. Hope it was worth the corkage fee. I'll handle finding the right pizza to go with that:

# In Ruby:
pairing_ruby[:zinfandel]
    # => 'hawaiian'

// In JS:
pairing_js.zinfandel;
    // => 'hawaiian'

Not a fan of white wines? Let's remove Chardonnay then.

# In Ruby:
pairing_ruby.delete(:chardonnay)

// In JS:
delete pairing_js.chardonnay;

Voila! And just like that, it's gone. Hey, I hear salad pizza is the new crave. Let's keep up with the times and add that to our menu:

# In Ruby:
pairing_ruby[:sauvignon_blanc] = 'salad'

// In JS:
pairing_js.sauvignon_blanc = 'salad';

In these simple examples, you can see how JS objects behave very similarly to Ruby hashes. There are the obvious syntax differences, but in principle, we are able to create and manipulate hashes/objects essentially the same way.

And now I gotta run because this code isn't going to cook the food for me. I hope these examples gave you some continuity in moving from Ruby to Javascript just as it did for me, and maybe spark the wine/food pairing curiousity in you.
