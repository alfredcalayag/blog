Block vs. Inline vs. Inline-Block. What's the difference and when do you want to use them? I'm a very visual learner, so for the purposes of "seeing it" let's play around with some divs, shall we? Let's use 3 divs set to 50px by 50px and assign them id's: 1, 2, and 3. i.e. our HTML text would look something like this:

< div id="1" >

< div id="2" >

< div id="3" >

In our CSS file, here's what we would assign to each div:

display:block--> Places each sequential div under the previous one forming a column-like effect.

2
3

display:inline--> Lines up each sequential div to immediately to the right of the previous div. Sort of looks like a side-by-side effect, but does not create a line-break at the end of each div. In the case a div reaches the end of it's container, it will wrap onto a new line (you know, like when a typewriter reaches the end of a line?... i also hope you know what a typewriter is). Another way of looking at it is similar to a "word-wrap". Notice how there is no margin between div and each "hugs" up next to each other. It is also important to note that a width and height cannot be set in this configuration. See how it ignored my height and width that was set to 50px x 50px?

display:inline-block-->Gives a true side-by-side effect that places divs only to the left and right. This is different from display:inline in that it DOES NOT have the "wrapping" feature. Also notice that a margin is automatically created between each div.
